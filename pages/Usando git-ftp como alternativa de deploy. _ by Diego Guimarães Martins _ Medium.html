Usando git-ftp como alternativa de deploy.

Para você, assim como eu, que gostaria de usar o versionamento do git e upar os arquivos no seu servidor de maneira fácil.

Diego Guimarães Martins

·

Bem, meus caros, esse é o meu primeiro artigo aqui no Medium e resolvi fazer sobre um assunto de algo que me salva bastante. Sim, fazer o deploy com outras formas e utilizando outras linguagens não é o meu forte. Já vi alternativas com Django e Python (sei lá se foi isso mesmo), além de algumas alterações no arquivo hook do Git e sendo bem franco, eu não peguei muito a ideia.

Então um belo dia…

Acabei me deparando com um artigo feito pelo Jean Suissa no blog dele, o Nomadev, cujo o link do artigo eu não achei mas juro que foi lá, que ensinava a usar esse maravilhoso plugin do Git, o git-ftp. Ele utiliza ftp para fazer upload dos seus arquivos para o servidor web de sua preferência, bastando apenas fazer pequenas configurações nele (acredite, são apenas 3!). O que mais pode dar trabalho é instalá-lo se você não for usuário do Linux.

Como então?

Irei aqui ensinar a instalação e configuração para Windows e Linux (se você é usuário de Mac, me perdoe). No repositório dos caras tem um manual de instalação que é super easy, segue o link: manual aqui.

O que é preciso?

  • Ter o Git instalado no S.O. que você utiliza;
  • Entender um pouco de comandos via Terminal;
  • Ter um Web Hosting com suporte ao Git e Ftp.

Instalação no Windows

Seguindo o próprio manual, a instalação nesse S.O. é simples mas, eu só testei a opção que eles dizem ser a recomendada, que é a seguinte:

  • Imaginando que você já tenha o Git no seu Windows, abra o terminal Git Bash (que é o terminal que vem na instalação do Git para Windows) em modo administrador, ou seja, clique com o botão direito do mouse em cima do arquivo “git-bash.exe” e escolha a opção “abrir como administrador”.
  • Navegue, pelo terminal, até o caminho onde você instalou Git que por padrão é
`C:\Program Files (x86)\Git
  • copie e cole no terminal o seguinte código:
curl https://raw.githubusercontent.com/git-ftp/git-ftp/master/git-ftp > /bin/git-ftp
  • quando o download terminar, copie e cole:
chmod 755 /bin/git-ftp

Se deu tudo certo, no caminhoC:\Program Files (x86)\Git\usr\bin deve existir um arquivo chamado git-ftp. Pronto, já tem o plugin instalado. Ou você pode executar git-ftp -vno terminal e aparecerá a versão instalada.

Instalação no Linux (usando Ubuntu)

  • Abra o terminal e digite o comando abaixo:
sudo apt-get install git-ftp

Pronto! Seu git-ftp já estará instado. Difícil, né?

Configurando a brincadeira

Feito uma das instalações acima, a partir de agora a coisa é igual para todos os S.O. Utilizando o terminal, vá até a pasta onde estão os arquivos que você gostaria de enviar para seu servidor. Por exemplo, c:\wamp\htdocs\www\meu-projeto, e uma vez lá, faça os procedimentos normais do Git de add e commit e agora espere.

O segredo vem aqui!

É agora na hora de dar um push que a brincadeira fica interessante, mas antes disso, lembra que eu disse que com 3 passos se configura esse plugin, pois bem:

  • configure um usuário digitando:
git config git-ftp.user ‘nomeDoUsuarioDoFTP’
  • configure o caminho ftp do seu servidor:
git config git-ftp.url ftp.example.net/public_html/opcional1/opicional2

— é interessante informar a pasta raiz do seu server, ex: public_html.
— você pode informar quantos subdiretórios quiser: /opcional1/opcional2

  • configure uma senha:
git config git-ftp.password ‘suaSenhaDoFTP’

e Finish Him!
Sim, meu(minha) jovem, é isso mesmo, acabou a configuração.

Agora a melhor parte

Duas coisas apenas pra dizer:

  • Ao enviar o projeto pela primeira vez ao seu servidor você deve executar:
git ftp init

— isso faz com que no servidor seja criado uma pasta .git e o head do commit seja preenchido com a versão atual do seu branch.

  • Da segunda vez diante, o comando é:
git ftp push branchQueVoceEstiverTrabalhando

Concluindo

Pronto, agora toda vez que você fizer alterações no projeto e der um git ftp push, somente os arquivos modificados serão upados para o seu servidor web.
Isso é lindo demais!!! :D

Espero que tenham gostado e que isso seja útil para muita gente assim como é para mim! Grato a todos, até a próxima.
